Apr 26, 2026
🎾
Fonseca — 3-hour live buffer to prevent mid-match disappearance
Events were being deleted by the Swift binary as soon as a match started (startDate ≤ now). Fixed by keeping events in the fetch window for 3 hours after start time so the calendar entry stays visible while a match is live.

FURIA — 4-hour live buffer for CS2 Bo3 matches
Same disappearing-event fix applied to FURIA. CS2 Bo3 matches can run long, so the buffer is 4 hours instead of 3.

⚡
Two-tier fast update — daily unless game today/tomorrow
Tennis and FURIA jobs now run once per day when no match is imminent, and switch to every 5 minutes when there's a game today or tomorrow. Saves API calls without losing precision near game time.

Feature Fonseca FURIA CS2 LaunchAgent

LaunchAgent split into 3 separate jobs
Separated into --fast (every 5 min, tennis + FURIA), --soccer (Sundays 09:00, Cruzeiro only), and --worldcup (hourly from Jun 24). Each job uses selective deletion meta so it never touches the other sports' events.
